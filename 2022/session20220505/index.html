Conflicting Refactorings

  •  
  • Programming in small groups
  • Free Language

The dilemma:

  •  
  • Two smells:
    • Message Chain
    • Middle Man
  • To get rid of one, you introduce the other

Message Chain:

  •  
  • Some method in object A calls a method defined on an object C in a field of an object B.
  • Example: 
    manager = person.department.manager;
  • This increases coupling & violates the Law of Demeter.
  • To get rid of this smell, hide the delegate object.

Middle Man (Person?)

  •  
  • The only thing an object does is forward calls from one object to another one.
  • To get rid of this smell, call the target object directly, and remove the then useless middle man.
